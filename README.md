# Голографический 3D-Лабиринт с управлением по гироскопу (для Holobox)

Этот проект представляет собой интерактивную 3D-игру лабиринт, адаптированную для вертикального 4K-экрана Holobox (соотношение сторон 9:16) и управляемую в реальном времени с помощью гироскопа смартфона.

Полная техническая архитектура и пайплайн описаны в файле [PROJECT_DOCUMENTATION.md](file:///c:/Users/RocketPC/Desktop/TheMostCreative/phone%20gyro%20control/PROJECT_DOCUMENTATION.md).

---

## 🛠️ Системные требования и зависимости

1. **Компьютер с Holobox**:
   - Установлен Node.js (версия 18 или новее).
   - Компьютер и смартфон должны находиться в **одной локальной сети Wi-Fi**.

2. **Смартфон**:
   - Любой современный смартфон на iOS (Safari) или Android (Chrome) с поддержкой датчика положения (акселерометра/гироскопа).

## 🚀 Быстрый запуск

### Шаг 1. Запуск сервера
В папке проекта откройте терминал (PowerShell или CMD) и выполните:

```bash
npm run dev
```

Сервер автоматически определит ваш локальный IP-адрес и запустится на порту `3000`. Вы увидите вывод в консоли:
```text
----------------------------------------------------------------
Server running at:
  - Local:      http://localhost:3000/
  - Network:      http://192.168.1.100:3000/
  - Controller:   http://192.168.1.100:3000/controller.html
----------------------------------------------------------------
```

### Шаг 2. Запуск игры на Holobox (ПК)
Откройте браузер на ПК (подключенном к экрану Holobox) и перейдите по адресу:
`http://localhost:3000/` (разверните вкладку на весь экран, нажав `F11`).

### Шаг 3. Подключение смартфона
1. На экране Holobox отобразится стартовый экран с **QR-кодом** и ссылкой.
2. Отсканируйте QR-код камерой смартфона.
3. На смартфоне откроется страница контроллера.
4. Положите смартфон ровно на ладонь (экраном вверх) и нажмите кнопку **"Активировать"**.
   - *Примечание для iOS (iPhone)*: Браузер Safari запросит разрешение на доступ к датчикам движения. Подтвердите его.
5. После активации стартовый экран на Holobox мгновенно скроется, и начнется первый лабиринт (Уровень 02).

## 🎮 Игровой процесс

- **Управление наклоном**: Наклоняйте смартфон на ладони. Лабиринт синхронно наклоняется в 3D-пространстве, заставляя шар реалистично катиться под действием силы тяжести.
- **Калибровка**: Если при ровном положении телефона лабиринт накренен, нажмите кнопку **"Калибровать"** на смартфоне или **"Сбросить Угол"** на экране ПК.
- **Сейвы и Финиш**: 
  - На месте финиша каждого лабиринта находится золотой вращающийся 3D-объект сейва (`save.fbx`), подсвеченный снизу неоновым розовым светом.
  - При касании сейва шар **плавно останавливается за 0.1 секунды**, воспроизводится победный аккорд, а сейв взлетает вверх к счетчику, увеличиваясь в размерах, и замирает там в горизонтальном ряду.
  - После этого уровень автоматически переключается на следующий (всего 6 уровней: `labirint2` – `labirint7`).
  - При сборе всех 6 сейвов игра считается пройденной, показывая экран победы.

## 📁 Структура проекта

- `server.js` — Локальный Express + Socket.io сервер с интеграцией Vite-middleware для HMR.
- `index.html` / `src/desktop.ts` — Десктопная часть игры: рендеринг Three.js, физика Rapier.js (WASM), Web Audio API синтезатор звуков, динамическое сопоставление координат финиша.
- `controller.html` / `src/controller.ts` — Контроллер для смартфона со считыванием датчиков `deviceorientation` и Socket.io клиентом.
- `public/source/new/` — Папка с 3D-моделями уровней (`labirint2.fbx` – `labirint7.fbx`).
- `public/source/save.fbx` — 3D-модель сейва (масштабирована в 20 раз меньше оригинала).
- `vite.config.js` — Конфигурация Vite для компиляции многостраничного приложения.

## 💎 Оптимизация для Holobox (Голографический эффект)

1. **Прозрачный черный фон**: Задний план полностью выкрашен в глубокий `#000000`. На экранах Holobox черный цвет отображается как прозрачный, создавая эффект парения лабиринта внутри куба.
2. **Точечный неон**: Свечение PointLight на старте и финише подчеркивает трехмерный объем и форму стенок.
3. **PBR Материалы**: Лабиринт использует высококачественные PBR текстуры (`Normal`, `Roughness`, `Metallic`), красиво реагирующие на источники света при изменении наклона.
4. **Ортографическая камера**: Исключает перспективные искажения, делая лабиринт идеально плоским сбоку и объемным при наклонах.
