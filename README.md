# Holobox Gyro Labyrinth

Интерактивная 3D-игра для вертикального Holobox: один гость сканирует одноразовый QR-код, разрешает смартфону доступ к датчикам движения и управляет наклоном лабиринта. Игра состоит из 4 уровней; за 60 секунд нужно собрать 4 сейва.

Продакшен размещается целиком на Render.com. Holobox и смартфоны открывают один публичный HTTPS-адрес, поэтому им нужен доступ в интернет, но не обязательно находиться в одной Wi-Fi-сети.

## Быстрый старт разработки

Требуется Node.js 22.12 или новее.

```bash
npm ci
npm run dev
```

Локальный сервер создаёт самоподписанный HTTPS-сертификат и открывается на `https://localhost:3000`. QR автоматически выбирает физический Wi‑Fi/LAN-адаптер; при необходимости адрес можно зафиксировать через `LOCAL_IP` в `.env`. На первом открытии смартфону потребуется подтвердить локальный сертификат.

VPN-клиент должен разрешать доступ к локальной сети. Если AmneziaVPN/WireGuard перехватывает маршрут Wi‑Fi-подсети, телефон не сможет открыть локальный QR даже при правильном IP — включите в VPN опцию доступа к LAN либо отключите VPN на время локального теста. В рабочем сценарии открывайте и Holobox, и телефон через публичный HTTPS-домен Render: локальные сертификаты и маршруты тогда не участвуют.

Для локальной генерации скопируйте `.env.example` в `.env` и задайте в нём `OPENROUTER_API_KEY`. Реальный `.env` исключён из Git и автоматически читается только при локальном запуске; Render продолжает получать секреты из своего Environment.

Важно: запускайте именно `npm run dev`, а не `vite`, Live Server или отдельную конфигурацию IDE. Express, Socket.IO и Vite должны работать одним Node.js-процессом. Быстрая проверка правильного процесса: `https://localhost:3000/health` возвращает JSON. Если страница открывается, но `/api/generate-image` или `/api/server-info` отвечает `404`, остановите отдельный frontend-сервер и перезапустите `npm run dev`.

Для проверки production-сборки:

```bash
npm run preview:prod
```

Основные команды:

- `npm run verify` — синтаксис, TypeScript, тесты и production build;
- `npm test` — протокол, координатор сессий, soak и HTTP/WebSocket smoke-тест;
- `npm start` — запуск уже собранного production-сервера;
- `npm run build` — сборка только нужных HTML, JS, CSS и runtime-ассетов.

## Деплой на Render

В корне находится `render.yaml`. Создайте в Render новый Blueprint из репозитория. Конфигурация уже задаёт:

- один постоянно работающий Starter-инстанс;
- `npm ci && npm run verify` как build command;
- `npm start` как start command;
- `/health` как health check;
- persistent disk `/var/data` для обезличенного журнала сессий;
- автоматически сгенерированные `OPERATOR_TOKEN` и `KIOSK_TOKEN`.

Render сам передаёт приложению `PORT` и `RENDER_EXTERNAL_URL`. Вручную задавать публичный URL или поднимать отдельный туннель не требуется.

После деплоя Holobox открывает обычный корневой URL сервиса. В ответ сервер выдаёт подписанную host-only kiosk-сессию в cookie с флагами `HttpOnly`, `Secure` и `SameSite=Strict`; `KIOSK_TOKEN` остаётся только в Render Environment и не попадает в JavaScript. Старый адрес вида `https://phone-gyro-control.onrender.com/#kiosk=<KIOSK_TOKEN>` поддерживается как резервный способ активации. Смартфон получает обычный публичный домен из QR-кода, без kiosk-секрета.

Важно: не увеличивайте `numInstances` выше 1. Активная пара «Holobox ↔ телефон» хранится в памяти процесса. Горизонтальное масштабирование потребует общего хранилища состояния и Socket.IO-адаптера.

## Игровой цикл

1. Гость нажимает «Начать»: стартовые надписи расходятся, а фон остаётся без перезагрузки.
2. Гость выбирает Англию, Францию, Испанию или Аргентину. Выбранный FBX-мяч делает полный оборот, увеличивается и переходит в фото-состояние.
3. На белом экране гость снимает фото встроенной камерой Holobox, при необходимости переснимает и подтверждает кадр. Снимок можно сделать кнопкой или удержав жест «большой палец вверх» примерно полсекунды.
4. После подтверждения приложение собирает промпт выбранной сборной с жёстким требованием сохранить пол и внешность и отправляет фото как identity-reference через защищённый same-origin API.
5. Только после подтверждения фото сервер выдаёт QR с одноразовым токеном на 90 секунд. Первый телефон, активировавший QR, получает эксклюзивную сессию.
6. После подключения телефона Holobox показывает белый экран загрузки. Когда лабиринт готов, появляется liquid-glass подсказка; трёхсекундный отсчёт начинается только после кнопки «СТАРТ».
7. Каждый собранный сейв на 2 секунды замораживает физику и таймер, слегка размывает лабиринт и показывает прогресс. В HUD собранные предметы отмечаются иконкой `source/save.svg`.
8. Готовый сгенерированный образ можно открыть кнопкой над лабиринтом; полноэкранный просмотр ставит игру на паузу и продолжает её после закрытия.
9. Короткий разрыв связи до 5 секунд ставит игру на паузу и позволяет тому же телефону восстановить управление.
10. После результата телефон отключается, а Holobox через 7 секунд возвращается к полному стартовому сценарию для следующего гостя.

Игровой звук отключён. Загрузка ассетов, подсказка, countdown, паузы между лабиринтами и просмотр образа не расходуют 60 секунд игрового времени.

## Фото и генерация

Камера работает только на публичном HTTPS-домене Render или на разрешённом локальном origin. Снимок держится в памяти вкладки, не записывается в JSONL-журнал и очищается при новом госте.

Распознавание `Thumb_Up` выполняется локально в браузере через MediaPipe Gesture Recognizer. Модель и SIMD/no-SIMD WASM лежат в `/mediapipe`, загружаются лениво только после включения камеры и не требуют CDN. Ручная кнопка остаётся доступной, если MediaPipe не инициализировался.

Для реальной генерации задайте в Render Environment секрет `OPENROUTER_API_KEY`. Модель уже задана в `render.yaml` через `OPENROUTER_IMAGE_MODEL=google/gemini-3.1-flash-image` и при необходимости может быть заменена без изменения кода.

Браузер не обращается к OpenRouter напрямую и не получает API-ключ. Node.js на Render принимает защищённый `POST /api/generate-image` размером до 12 МБ, проверяет `KIOSK_TOKEN` и вызывает OpenRouter Images API с исходным фото как reference, вертикальным соотношением `9:16` и `resolution: 4K`. Фото, промпт и результат не пишутся на диск и не попадают в журнал; HTTP-ответы помечены `no-store`. Таймаут внешнего вызова — 480 секунд и настраивается через `OPENROUTER_TIMEOUT_MS`. Сетевые сбои, timeout, rate limit, недостаток средств, неверный reference и ошибки провайдера получают разные безопасные коды и `requestId`; последняя диагностика видна в защищённом `/api/operator/status`. Если ключ не настроен или OpenRouter временно недоступен, лабиринт всё равно запускается.

## Редактор экранов и 3D-сцены

Шестерёнка открывает встроенный редактор всех основных DOM-элементов, камер и 3D-корней. Для DOM доступны позиция, раздельный масштаб и наклоны XYZ; для Three.js-объектов — позиция XYZ, масштаб XYZ и углы XYZ. Отдельно настраиваются камеры стартового экрана, выбора команды и лабиринта, а также плотность/размер задней теневой плоскости и положение теневого света. Значения сохраняются только в `localStorage` текущего Holobox-браузера; кнопки «Сбросить элемент» и «Сбросить всё» возвращают исходные параметры.

Все новые `Mesh` автоматически получают `castShadow`/`receiveShadow` при следующем аудите сцены. Новый 3D-корень автоматически появляется в редакторе, если он добавлен непосредственно в сцену с уникальным `name`; для вложенных объектов задайте `object.userData.editorId` и необязательный `editorLabel`.

## Операторская панель

Панель находится по адресу `/operator.html`. Введите значение `OPERATOR_TOKEN` из Render Environment. Панель показывает состояние станции, возраст последней телеметрии и счётчики сессий; из неё можно вернуть станцию к QR или запросить калибровку.

`/health` доступен без токена и предназначен для Render. Операторские API защищены `OPERATOR_TOKEN`, а регистрация Holobox и выдача QR — отдельным `KIOSK_TOKEN`.

## Структура

- `server.js` — HTTP, WebSocket, health и operator API;
- `server/sessionCoordinator.js` — владение контроллером и жизненный цикл сессий;
- `shared/protocol.js` — единые события и строгая проверка входных данных;
- `src/desktop.ts` — Three.js/Rapier и UI Holobox;
- `src/experience/` — выбор сборной, FBX-анимация и безопасная сборка промпта для фотографии;
- `src/controller.ts` — разрешения датчиков, reconnect и телеметрия смартфона;
- `src/game/` — конфигурация уровней, игровой таймер, ассеты и освобождение ресурсов;
- `src/operator.ts` — панель дежурного;
- `tests/` — модульные, soak и интеграционные тесты;
- `PROJECT_DOCUMENTATION.md` — архитектура и протокол;
- `PROJECT_GUIDE.md` — эксплуатационный чек-лист мероприятия.

Функционал кампании и продуктовые интеграции в эту инфраструктурную конфигурацию намеренно не включены.
