# Документация проекта: Holobox Gyro Labyrinth 3D

Проект представляет собой трехмерную игру-лабиринт с физической симуляцией, разработанную специально для вертикальных экранов 4K Holobox (соотношение сторон 9:16). Управление наклоном лабиринта осуществляется в реальном времени с помощью смартфона, использующего гироскоп.

---

## 1. Цель проекта и суть игры
* **Цель**: Создать интерактивный игровой стенд для Holobox, где пользователь катит трехмерный шар по лабиринту от старта (бирюзовая зона) до финиша (розовая зона), наклоняя свой смартфон как физический балансир.
* **Суть**: Игрок держит смартфон горизонтально в руке. Наклоны телефона вперед/назад и влево/вправо транслируются в наклоны лабиринта на экране. Шар катится под действием гравитации, подчиняясь законам физики. Игра сопровождается процедурным аудио-фидбеком (гул качения, стуки ударов об стены, фанфары победы).

---

## 2. Стек технологий и инструменты
* **3D-графика**: [Three.js](https://threejs.org/) (веб-рендерер WebGL с поддержкой теней PCFSoftShadowMap).
* **Физический движок**: [Rapier.js WASM](https://rapier.rs/) (высокопроизводительный физический движок, скомпилированный в WebAssembly).
* **Сетевой мост (WebSocket)**: [Socket.io](https://socket.io/) (двусторонний обмен телеметрией с ультра-низкой задержкой).
* **Серверная часть**: [Node.js Express](https://expressjs.com/) с автоматической генерацией 2048-битных SSL-сертификатов (SAN) для защищённого контекста HTTPS.
* **Сборщик и HMR**: [Vite](https://vite.dev/) с поддержкой TypeScript.
* **Аудио**: [Web Audio API](https://developer.mozilla.org/en-US/docs/Web/API/Web_Audio_API) (полностью процедурный синтез звуков на лету, не требующий загрузки тяжелых аудиофайлов).

## 3. Архитектура и реализованный пайплайн

### 3.1. Сетевое сопряжение (Mobile ◄─► Server ◄─► Desktop)
1. **HTTPS Сервер**: Генерирует временный SSL-сертификат для локального IP (например, `10.8.1.1`) на лету, чтобы мобильный браузер разрешил доступ к API гироскопа (`DeviceOrientationEvent`), который работает **только в защищённом контексте (HTTPS)**.
2. **Сопряжение по QR-коду**: ПК выводит на экран Pairing-карту со сгенерированным QR-кодом. Смартфон сканирует его и подключается к веб-сокету.
3. **Обмен телеметрией**: Смартфон считывает углы `beta` (тангаж) и `gamma` (крен) и шлёт их на сервер с частотой 60 Гц. Сервер пересылает их десктопному клиенту.

### 3.2. Трехмерный мир и Физика
1. **Нормализация масштаба (SI Units)**: Оригинальная 3D-модель лабиринта имела гигантские размеры (3.5 километра по ширине). Мы масштабируем её до реалистичных **12.0 метров**, что соответствует идеальному диапазону работы физического движка (0.1–10.0м). Радиус шара составляет **26 см**.
2. **Статический коллайдер (Trimesh)**: Лабиринт в физическом мире зафиксирован неподвижно (`RAPIER.RigidBodyDesc.fixed()`), чтобы избежать погрешностей расчета контактов при движении сложных полигональных сеток.
3. **Виртуальная гравитация (Tilt)**: Вместо физического вращения лабиринта в физическом мире мы **наклоняем вектор гравитации** (`physicsWorld.gravity`) пропорционально углам наклона смартфона. Математически это абсолютно эквивалентно, но дает **100% стабильность столкновений** без риска выпадения или прохождения шара сквозь стены.
4. **Визуальное вращение (`mazeContainer`)**: Чтобы игрок видел наклон, все визуальные элементы (сетка лабиринта, стартовые огни, финишный диск, плоскость теней) вложены в единый контейнер `mazeContainer` в Three.js, который вращается на углы наклона. Шар визуально смещается относительно центра вращения на ту же матрицу поворота.

### 3.3. Камера
Камера расположена спереди лабиринта. Клавиши **Стрелка Вверх** и **Стрелка Вниз** на ПК позволяют изменять угол наклона камеры (в диапазоне от **5°** до **85°**), давая возможность смотреть на лабиринт более полого (3D-эффект) или строго сверху (классический 2D-вид).

### 3.4. fallbacks (Аварийные режимы)
* **Клавиатура на ПК**: Клавиши **W, A, S, D** позволяют тестировать наклоны лабиринта без телефона. При отпускании клавиш лабиринт плавно возвращается в горизонт.
* **D-pad на Смартфоне**: На экране контроллера добавлены кнопки-стрелки ▲, ▼, ◀, ▶. Если гироскоп на телефоне недоступен, пользователь может управлять шаром, нажимая кнопки пальцем.

## 4. Решенные проблемы и баги

| Проблема | Причина | Решение |
| :--- | :--- | :--- |
| **Гироскоп не работал на телефоне** | Ограничение безопасности браузеров: `DeviceOrientation` требует HTTPS. | Написан [server.js](file:///c:/Users/RocketPC/Desktop/TheMostCreative/phone%20gyro%20control/server.js), генерирующий SSL-сертификаты с SAN для локального IP ПК. |
| **Черный экран / бесконечный сброс** | Мяч падал с высоты 140 метров сквозь отсутствующий пол старого лабиринта и улетал в бездну, вызывая постоянный перезапуск и калибровку в 0. | Добавлен контроль CCD, спавн перенесен вплотную к полу. Впоследствии добавлена новая модель со встроенным полом. |
| **Шар проходил сквозь стены при наклоне** | Центр вращения визуальной сетки (`-center`) не совпадал с центром физического мира `(0,0,0)`. | Создана иерархия `mazeContainer` (отцентрирована в `(0,0,0)`), вращающаяся строго синхронно с трансформациями шара. |
| **Шар застывал и не катался** | Движок Rapier переводил шар в режим сна (Sleep) при остановке. Изменение глобальной гравитации не будило его. | Для RigidBody шара отключено засыпание: `.setCanSleep(false)`. |
| **Лабиринт оставался старым после замены файла** | Агрессивное кэширование браузером файлов FBX и текстур. | Добавлен механизм Cache Busting (загрузка с временным штампом `?v=Date.now()`). |

## 5. Текущие проблемы и открытые вопросы

1. **Размещение файлов**:
   * **КРИТИЧЕСКИ ВАЖНО**: Новые модели лабиринта (`labirint.fbx`) и текстуры необходимо загружать **ТОЛЬКО** в папку **`public/source/`** и **`public/textures/`** соответственно.
   * Папка `dist/` — это папка **продакшн-билда**. Команда `npm run build` (или `vite build`) **копирует** файлы из `public/` → `dist/`, полностью перезаписывая содержимое `dist/`.
   * **Инцидент**: Новый файл `labirint.fbx` был загружен в `dist/source/`, после чего `npm run build` перезаписал его старым файлом из `public/source/`. Новый файл был утерян.
   * **Правило**: Всегда загружайте ассеты в `public/`, никогда в `dist/`.

2. **Первоначальная калибровка**:
   * Для корректной работы гироскопа пользователь должен держать смартфон строго горизонтально в момент нажатия кнопки **«Калибровка»**. При сильном отклонении во время калибровки углы управления могут упереться в лимит (клиппинг).

3. **Текущая модель лабиринта**:
   * Используется старая модель `labirint.fbx` (1,024,748 байт от 25.06.2024) **без встроенного пола**.
   * В коде принудительно добавлен физический пол (20 см кубоид-коллайдер) под основанием лабиринта.
   * Когда новая модель (с полом) будет загружена в `public/source/`, этот коллайдер можно будет убрать из кода.
